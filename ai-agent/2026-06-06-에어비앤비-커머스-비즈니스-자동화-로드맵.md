### Context
* 다각화된 수익 모델(에어비앤비, 쇼핑몰, 오프라인 판매 등)을 구축하기 위해 사업의 우선순위를 설정하고 기술적 자동화 방안을 모색함.
* 가장 먼저 착수할 에어비앤비(Airbnb) 사업의 운영 효율을 극대화하기 위해 AI 에이전트를 활용한 고객 응대 자동화 로직을 설계함.
* 이후 쇼핑몰 및 오프라인 판매로 확장 시 데이터 일관성을 유지할 수 있는 통합 워크플로우를 준비함.
* 반복적인 업무를 자동화하여 운영자의 개입을 최소화하고 확장성 있는 비즈니스 구조를 구축하는 것이 핵심 목표임.

### Core
* n8n의 AI Agent 노드를 활용하여 에어비앤비 예약 문의 및 숙소 안내 업무를 자동화하는 로직을 구성함.
* 예약 플랫폼의 Webhook 또는 이메일 파싱(Email Parsing)을 통해 수신된 메시지를 AI가 분석하고 적절한 답변을 생성함.
* 사용 예시(Pseudo code/Logic):
```json
{
  "agent_type": "Tool-Use Agent",
  "knowledge_base": "House_Manual_RAG",
  "tools": [
    "Google_Calendar_Check",
    "Airbnb_Messaging_API",
    "Slack_Notification"
  ],
  "prompt": "고객의 문의 내용이 숙소 이용 규칙이나 위치 정보인 경우 RAG를 통해 답변하고, 예약 변경 등은 도구를 호출하세요."
}
```
* 주요 자동화 단계:
* 수신된 메시지의 의도 파악(단순 문의, 긴급 상황, 예약 변경 등) 및 분류.
* RAG(Retrieval-Augmented Generation)를 기반으로 한 숙소 매뉴얼 정보 검색 및 답변 생성.
* 긴급 상황 발생 시 운영자에게 즉각적인 알림(Slack/Discord) 발송.
* 쇼핑몰 확장 시 n8n의 Shopify/WooCommerce 노드를 결합하여 주문 처리 자동화 연계.

### Insight
* 에어비앤비 공식 API는 접근 권한이 제한적이므로, Guesty나 Hospitable 같은 PMS(Property Management System)의 API를 n8n의 HTTP Request 노드로 호출하는 방식이 기술적으로 안정적임.
* AI 에이전트 도입 시 할루시네이션(Hallucination) 방지를 위해 반드시 벡터 DB에 숙소 매뉴얼을 저장하고 'Knowledge'로 연결해야 고객 신뢰도를 유지할 수 있음.
* 쇼핑몰과 오프라인 판매로 확장 시 재고 관리 시스템(WMS)과 AI 에이전트를 연결하여 통합 재고 모니터링 환경을 구축하는 것이 유리함.
* 초기 단계에서는 완전 자동화보다는 '초안 생성 후 운영자 승인' 프로세스(Human-in-the-loop)를 도입하여 응답의 정확도를 검증하는 과정이 필수적임.
