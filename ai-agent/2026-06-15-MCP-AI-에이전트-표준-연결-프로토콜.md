### Context
최근 LLM 에이전트의 활용이 늘어나면서 다양한 데이터 소스와 도구를 연결하는 표준화된 인터페이스의 필요성이 대두되었습니다. Model Context Protocol(MCP)은 마치 하드웨어의 USB-C 포트처럼, 서로 다른 AI 모델과 데이터 시스템 간의 연결을 단순화하고 효율성을 극대화하기 위해 설계된 개방형 프로토콜입니다.

### Core
MCP는 클라이언트-서버 아키텍처를 기반으로 하며, 개발자가 일일이 도구 호출(Tool Calling) 로직을 새로 짤 필요 없이 규격화된 방식으로 리소스를 공유할 수 있게 합니다.

* **핵심 구조**: 호스트 어플리케이션(예: Claude Desktop), MCP 서버(데이터/기능 제공), MCP 클라이언트(서버와 통신)
* **주요 기능**:
  * 리소스(Resources): 파일 내용, 데이터베이스 조회 결과 등 읽기 전용 데이터 제공
  * 프롬프트(Prompts): 사전 정의된 프롬프트 템플릿 공유
  * 도구(Tools): 실행 가능한 함수 및 외부 API 연동 기능 제공
* **통신 방식**: JSON-RPC를 기반으로 하여 프로그래밍 언어에 구애받지 않고 유연한 통합 지원

### Insight
MCP는 AI 에이전트 구축 시 파편화된 통합 과정을 하나로 묶어주는 '기술적 허브' 역할을 합니다. 기존에는 각 서비스마다 전용 커넥터를 개발해야 했으나, MCP를 통해 한 번의 구현으로 다양한 LLM 환경에 즉시 적용할 수 있는 확장성을 확보하게 되었습니다. 이는 에이전트의 데이터 접근성을 비약적으로 향상시키며 개발 주기를 단축시키는 핵심 요소가 될 것입니다.
