### Context
과거 AI 에이전트 개발 시 가장 큰 병목 현상은 각기 다른 데이터 소스(Google Drive, Slack, Local DB 등)와 모델을 연결하기 위해 매번 개별 커넥터를 개발해야 한다는 점이었습니다. 이러한 파편화는 유지보수 비용을 증가시키고 에이전트의 확장성을 저해했습니다. Model Context Protocol(MCP)은 하드웨어의 USB-C 표준처럼 AI 모델과 데이터 도구 간의 통신 방식을 표준화하여, 한 번의 구축으로 다양한 환경에서 에이전트가 데이터를 즉시 활용할 수 있게 설계된 개방형 프로토콜입니다.

### Core
MCP 아키텍처는 모델이 데이터를 안전하고 효율적으로 읽어올 수 있도록 세 가지 핵심 계층으로 구성됩니다. 아래는 로컬 SQLite 데이터베이스를 MCP 서버로 등록하여 에이전트가 접근할 수 있도록 설정하는 예시 코드입니다.

*   MCP Host: 사용자와 상호작용하는 프로그램 (예: Claude Desktop, 에이전트 런타임)
*   MCP Client: 호스트 내에서 서버와 보안 연결을 유지하는 인터페이스
*   MCP Server: 실제 데이터 도구나 API 노출 (예: SQLite, GitHub 커넥터)

```json
{
  "mcpServers": {
    "sqlite": {
      "command": "uvx",
      "args": [
        "mcp-server-sqlite",
        "--db-path",
        "/Users/user/data/knowledge_base.db"
      ]
    }
  }
}
```

### Insight
MCP의 도입은 단순한 연결성 개선을 넘어 AI 에이전트 생태계에 다음과 같은 기술적 변화를 가져옵니다.

*   범용 연결성(Universal Connectivity): 특정 에이전트 프레임워크에 종속되지 않는 표준 커넥터를 통해 개발 생산성이 비약적으로 향상됩니다.
*   데이터 보안 강화: 로컬 데이터에 대한 접근 권한을 세밀하게 제어하며, 민감한 정보를 모델 학습에 직접 노출하지 않고 필요한 시점에만 컨텍스트로 제공합니다.
*   에이전트의 실시간성 확보: 정적 학습 데이터가 아닌 최신 데이터 소스에 직접 쿼리함으로써 응답의 정확도와 신뢰성을 극대화합니다.
*   커뮤니티 기반 확장: 오픈 소스 기반의 다양한 MCP 서버들이 공유됨에 따라 누구나 복잡한 연동 과정 없이 고급 기능을 에이전트에 추가할 수 있습니다.
