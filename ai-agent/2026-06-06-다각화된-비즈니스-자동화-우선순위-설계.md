### Context
* 다각화된 수익 모델(에어비앤비, 쇼핑몰, 오프라인)의 준비 과정에서 발생하는 리소스 분산을 막기 위한 순차적 실행 구조 설계.
* 운영 효율성을 극대화하기 위해 자동화 난이도가 낮고 시스템화가 용이한 에어비앤비를 첫 번째 실행 과제로 선정.

### Core
* 에이전트의 비즈니스 실행 순서 제어를 위한 상태 기반 워크플로우 로직 (Python)
```python
# 비즈니스 단계별 상태 관리 및 우선순위 큐 정의
workflow_stack = ["airbnb", "shopping_mall", "offline_sales"]

def trigger_next_business(current_phase):
    if current_phase in workflow_stack:
        next_idx = workflow_stack.index(current_phase) + 1
        return workflow_stack[next_idx] if next_idx < len(workflow_stack) else "Done"
```
* n8n의 Switch 노드를 활용하여 에어비앤비 자동화 시스템(예약 알림 및 체크인 안내) 구축 완료 시점을 감지하고 다음 비즈니스로 로직을 분기함.

### Insight
* 에어비앤비는 외부 API(Channel Manager) 연동이 성숙하여 초기 자동화 파이프라인 구축에 가장 적합한 모델임.
* 쇼핑몰 단계로 확장 시 물류 API 및 재고 관리 에이전트를 추가하여 운영 자동화의 깊이를 조절할 수 있음.
* 순차적 실행 방식은 시스템 리소스의 병목 현상을 방지하고 비즈니스 모델 간의 데이터 무결성을 유지하는 데 필수적임.
