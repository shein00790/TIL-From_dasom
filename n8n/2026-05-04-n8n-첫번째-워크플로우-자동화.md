### Context
*   사용자가 n8n을 활용하여 첫 번째 워크플로우 자동화를 구축한 경험에 대한 문서입니다.
*   Discord 메시지 수신을 시작으로, LLM 에이전트가 TIL 형식으로 문서를 변환하고, 이를 GitHub에 푸시하며, `README.md` 파일을 업데이트한 후 완료 메시지를 발신하는 일련의 과정을 자동화했습니다.

### Core
*   **Discord Trigger 노드**: Discord 채널에서 특정 메시지가 수신될 때 워크플로우를 시작합니다.
*   **Code 노드 또는 HTTP Request 노드**: 수신된 Discord 메시지를 기반으로 LLM 에이전트를 실행합니다. 이 에이전트는 메시지 내용을 분석하여 'thoughtbot/til' 스타일의 TIL(Today I Learned) 형식으로 변환합니다. (예: Markdown 형식의 파일 내용 생성)
*   **GitHub 노드 (파일 생성 및 커밋)**: 변환된 TIL 내용을 GitHub 저장소에 새로운 `.md` 파일로 커밋합니다. 파일명은 `YYYY-MM-DD-제목.md` 규칙을 따릅니다.
*   **GitHub 노드 (README.md 업데이트)**: 기존 `README.md` 파일을 가져와 새로 추가된 TIL 문서의 링크를 인덱스에 추가한 후, 업데이트된 `README.md` 파일을 다시 커밋합니다.
*   **Discord Send Message 노드**: 모든 과정이 성공적으로 완료되면, Discord 채널에 완료 메시지를 발신하여 사용자에게 알립니다.

### Insight
*   n8n 워크플로우를 처음 구축하면서, 각 노드의 '입력(Input)'과 '출력(Output)'을 정확히 이해하는 것이 워크플로우 설계의 핵심임을 깨달았습니다.
*   각 노드가 어떤 데이터를 받아 어떤 형태로 다음 노드에 전달하는지 파악하면 복잡한 자동화도 효율적으로 구성할 수 있습니다.
