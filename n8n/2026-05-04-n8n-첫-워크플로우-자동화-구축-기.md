### Context

첫 n8n 워크플로우 자동화를 구축하며 디스코드 메시지 수신부터 GitHub push까지 이어지는 과정을 경험했습니다. 이 자동화는 디스코드에서 특정 메시지를 트리거로 받아, Agent를 실행시키고, 해당 내용을 TIL 형식으로 변환하여 GitHub 저장소에 자동으로 push하는 과정을 포함합니다. 마지막으로 README.md 파일에 해당 내용을 인덱싱하고 완료 메시지를 발신하도록 구성했습니다.

### Core

*   **Trigger**: Discord 메시지 수신
*   **Agent Execution**: 수신된 메시지를 기반으로 Agent 실행
*   **TIL Conversion**: Agent 실행 결과를 TIL 형식의 Markdown으로 변환
*   **GitHub Push**: 변환된 TIL 파일을 지정된 GitHub 저장소에 push
*   **README Update**: `README.md` 파일에 새로 생성된 TIL 인덱스 추가
*   **Completion Notification**: 모든 과정 완료 후 Discord 채널에 완료 메시지 발신

*(참고: 실제 n8n 워크플로우 구성 노드 및 상세 설정은 사용자 입력에 포함되지 않아 구체적인 코드로 명시하지 못했습니다. 각 노드의 Input/Output 이해가 중요함을 강조합니다.)*

### Insight

n8n을 처음 사용해 보았는데, 생각보다 직관적이고 강력했습니다. 워크플로우 자동화 구축의 핵심은 각 노드가 어떤 `input`을 받아서 어떤 `output`을 생성하는지 명확히 이해하는 것이었습니다. 이 두 가지만 파악하면 복잡해 보이는 워크플로우도 쉽게 설계하고 구현할 수 있다는 것을 배웠습니다. 특히, 외부 서비스 연동이나 복잡한 로직 처리를 노드 조합만으로 구현할 수 있다는 점이 인상 깊었습니다.
