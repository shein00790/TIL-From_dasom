### Context
* 주식, 쇼핑몰 운영, 코딩 등 다양한 분야에서 발생하는 파편화된 기술 메모와 개인적 인사이트를 체계적으로 자산화할 필요성이 제기됨
* 수동으로 기록하던 메모를 n8n 워크플로우와 연결하여 중앙 데이터베이스(DB)에 자동으로 분류 및 저장함으로써 지식 탐색 효율을 극대화함

### Core
* Webhook 또는 HTTP Request 노드를 통해 외부 입력 데이터를 수집한 뒤, 데이터 특성에 따라 경로를 분기함
* Switch 노드를 활용하여 입력된 데이터의 키워드(예: '주식', 'n8n', 'Python')를 기준으로 해당 카테고리 DB 테이블로 할당함
* 데이터 무결성을 위해 데이터베이스 노드(Notion, Airtable, PostgreSQL 등)의 Upsert(Update if exists, else Insert) 기능을 사용하여 중복 데이터를 방지함
* 데이터 구조 예시:
```json
{
  "action": "upsert",
  "table": "personal_insights",
  "data": {
    "title": "n8n 자동 분류 로직",
    "category": "n8n",
    "content": "Switch 노드를 이용한 조건부 분기 설정 방법",
    "timestamp": "2026-06-01"
  }
}
```

### Insight
* 단순 저장을 넘어 데이터베이스의 검색 성능을 높이기 위해 태깅(Tagging) 자동화 로직을 추가하는 것이 권장됨
* 최신 기술 트렌드인 벡터 데이터베이스(Vector Database)와 n8n의 AI Agent 노드를 결합하면 단순 키워드 검색이 아닌 의미론적 검색(Semantic Search)이 가능해져 지식 활용도가 비약적으로 상승함
