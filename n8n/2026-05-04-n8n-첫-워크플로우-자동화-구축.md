### Context

첫 n8n 워크플로우 자동화를 구축하며 디스코드 메시지 수신 시 트리거가 발생하는 에이전트를 실행하고, 이를 TIL 형식으로 변환하여 GitHub에 푸시하는 과정을 자동화했습니다. 푸시 완료 후 README.md 파일에 인덱스를 추가하고 완료 메시지를 발송하는 과정을 구현했습니다.

### Core

*   **트리거**: 디스코드(Discord) 메시지 수신
*   **에이전트 실행**: 수신된 메시지를 기반으로 TIL 형식 문서 생성 및 GitHub 푸시
    *   TIL 형식 변환 로직 (내부 처리)
    *   GitHub 푸시 (Git 커밋 및 푸시 명령어 실행)
*   **README.md 업데이트**: 푸시된 TIL 문서의 인덱스를 README.md에 추가
*   **완료 메시지 발송**: 디스코드 채널에 자동화 완료 메시지 전송

### Insight

n8n을 처음 사용해 보았지만, 각 노드의 Input과 Output을 명확히 이해하면 복잡한 워크플로우도 비교적 쉽게 구축할 수 있다는 것을 알게 되었습니다. 특히, 각 노드가 어떤 데이터를 입력받아 어떤 형태로 변환하여 다음 노드로 전달하는지에 대한 이해가 워크플로우 설계의 핵심입니다.
