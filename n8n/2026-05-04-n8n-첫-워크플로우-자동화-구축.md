### Context
사용자는 n8n을 활용하여 첫 번째 워크플로우 자동화를 구축했습니다. 이 워크플로우는 Discord 메시지 수신을 시작으로, 기술 문서(TIL)를 생성하고 GitHub에 푸시하며, 최종적으로 완료 메시지를 발신하는 일련의 과정을 자동화합니다.

### Core
이 워크플로우는 다음과 같은 단계로 구성됩니다.

*   **Discord Trigger**: 특정 Discord 채널에서 메시지가 수신되면 워크플로우가 시작됩니다. 이 노드는 외부 이벤트를 감지하여 자동화 프로세스를 구동하는 역할을 합니다.
*   **AI Agent (LLM)**: 수신된 Discord 메시지를 분석하여 'thoughtbot/til' 스타일의 구조화된 기술 문서(TIL) 형식으로 변환합니다. 이 과정에서 LLM(Large Language Model) 기반의 에이전트가 활용되어 비정형 데이터를 정형화된 문서로 가공합니다.
*   **GitHub Node (Create/Update File)**: 변환된 TIL 문서를 GitHub 저장소에 새로운 파일로 생성하거나 기존 파일을 업데이트합니다. 이 노드는 파일 내용, 커밋 메시지, 브랜치 등 GitHub 관련 작업을 처리합니다.
*   **GitHub Node (Update README.md)**: 새로 추가된 TIL 문서의 인덱스를 `README.md` 파일에 자동으로 추가합니다. 이는 저장소의 문서 목록을 최신 상태로 유지하는 데 중요합니다.
*   **Discord Send Message**: 모든 자동화 과정이 성공적으로 완료되면, Discord 채널에 완료 메시지를 발신하여 사용자에게 작업 성공을 알립니다.

### Insight
n8n을 활용한 워크플로우 자동화는 각 노드의 '입력(Input)'과 '출력(Output)'을 정확히 이해하는 것이 핵심입니다. 각 노드가 어떤 데이터를 받아들이고, 어떤 형태로 다음 노드로 전달하는지를 파악하면 복잡한 자동화 로직도 쉽게 설계할 수 있습니다. 예를 들어, Discord Trigger 노드의 출력은 AI Agent 노드의 입력이 되고, AI Agent 노드의 출력은 GitHub 노드의 입력이 되는 식입니다. 이러한 데이터 흐름의 이해는 워크플로우의 안정성과 효율성을 크게 향상시킵니다.
