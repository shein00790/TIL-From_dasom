### Context

첫 n8n 워크플로우 자동화를 구축하며 디스코드 메시지 수신부터 GitHub 푸시까지 이어지는 전체 과정을 경험했습니다. 이 자동화는 디스코드에서 특정 메시지를 트리거로 받아, 해당 내용을 TIL(Today I Learned) 형식의 마크다운으로 변환하고, 이를 GitHub 저장소에 푸시한 후, README 파일을 업데이트하는 일련의 작업을 자동화하는 것을 목표로 합니다.

### Core

*   **트리거 노드**: 디스코드에서 특정 메시지를 수신하면 워크플로우가 시작됩니다.
*   **TIL 변환 노드**: 수신된 메시지 내용을 분석하여 TIL 형식의 마크다운 파일로 변환합니다.
*   **GitHub 푸시 노드**: 변환된 TIL 마크다운 파일을 지정된 GitHub 저장소에 푸시합니다.
*   **README 업데이트 노드**: 푸시된 TIL 파일의 인덱스를 README.md 파일에 추가합니다.
*   **완료 메시지 발신 노드**: 모든 작업이 완료되었음을 알리는 메시지를 디스코드로 발송합니다.

### Insight

n8n을 처음 사용해 보았는데, 각 노드의 입력(input)과 출력(output)을 명확히 이해하는 것이 워크플로우 구축의 핵심임을 깨달았습니다. 노드 간의 데이터 흐름을 파악하면 복잡한 자동화도 비교적 쉽게 설계할 수 있습니다. 이번 경험을 통해 n8n의 기본적인 노드 활용법과 워크플로우 설계 방식을 익힐 수 있었습니다.
