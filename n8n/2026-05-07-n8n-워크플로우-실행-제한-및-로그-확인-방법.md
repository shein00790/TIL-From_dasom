### Context
n8n 워크플로우를 처음 제작한 후 테스트 과정에서 실행 결과가 정상적으로 기록되는지 확인하기 어렵거나, 하루에 실행할 수 있는 횟수에 제한이 있는지에 대한 의문을 해결하기 위한 문서입니다. 특히 '퍼블리시(활성화)' 후 결과가 보이지 않는 현상의 원인을 기술적으로 분석합니다.

### Core
n8n에서 워크플로우 실행 여부를 확인하고 실행 횟수를 관리하기 위해 아래의 핵심 설정과 개념을 이해해야 합니다.

* **실행 데이터 저장 설정 (Save Execution Data)**
    * n8n은 데이터베이스 용량 관리를 위해 기본적으로 성공한 실행 기록을 저장하지 않도록 설정되어 있을 수 있습니다.
    * 설정 방법: 워크플로우 편집 화면 상단 메뉴 -> 'Settings' -> 'Save Successful Executions' 항목을 'True'로 변경해야 실행 이력(Executions) 탭에서 기록을 확인할 수 있습니다.

* **실행 제한 (Execution Limits)**
    * 셀프 호스팅(Self-hosted): 하드웨어 자원이 허용하는 한 실행 횟수나 퍼블리시 횟수에 제한이 없습니다.
    * n8n Cloud: 요금제 플랜에 따라 '월간 총 실행 횟수(Executions per month)' 제한은 있으나, 특정 워크플로우를 하루에 한 번만 실행해야 하는 일일 제한은 존재하지 않습니다.

* **테스트와 활성화의 차이**
    * Test Workflow (수동 실행): 워크플로우 편집기 하단의 버튼을 눌러 즉시 실행하는 것으로, 활성화 여부와 상관없이 무제한 테스트가 가능합니다.
    * Active (자동 실행): 우측 상단의 스위치를 켜서 활성화하면 트리거(Schedule, Webhook 등) 조건에 따라 자동으로 실행됩니다.

### Insight
n8n의 워크플로우 퍼블리시와 실행은 '하루에 한 번'과 같은 시간적 제약을 받지 않습니다. 만약 테스트 결과가 보이지 않는다면 이는 서비스 제한이 아니라 **실행 데이터 보존 설정(Save Data Policy)**의 문제일 가능성이 매우 높습니다. 초기 구축 시에는 모든 실행 데이터를 저장하도록 설정하여 각 노드의 입력과 출력값이 의도대로 흐르는지 검증하는 과정이 필수적입니다. 또한, 'Executions' 메뉴에서 실패(Error) 기록이 있는지 확인하여 트리거가 정상적으로 작동했는지 점검해야 합니다.
