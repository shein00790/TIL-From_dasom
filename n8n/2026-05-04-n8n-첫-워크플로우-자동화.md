### Context

첫 n8n 워크플로우 자동화를 통해 디스코드 메시지 수신 시 에이전트 실행, TIL 형식 변환, GitHub 푸시, README 업데이트 및 완료 메시지 발송까지 이어지는 전체 과정을 구현했습니다.

### Core

*   **트리거**: 디스코드 메시지 수신
*   **에이전트 실행**: 수신된 메시지를 기반으로 에이전트 로직 수행
*   **TIL 형식 변환**: 에이전트 실행 결과를 TIL(Today I Learned) 형식의 마크다운으로 변환
*   **GitHub 푸시**: 변환된 TIL 문서를 지정된 GitHub 저장소에 푸시
*   **README 업데이트**: 푸시된 TIL 문서의 인덱스를 README.md 파일에 추가
*   **완료 메시지 발송**: 모든 과정 완료 후 디스코드 채널에 완료 메시지 발송

### Insight

n8n을 처음 사용해 보았는데, 각 노드의 입력(input)과 출력(output)을 명확히 이해하는 것이 워크플로우 설계의 핵심임을 알게 되었습니다. 이 두 가지만 정확히 파악하면 복잡한 자동화도 비교적 쉽게 구축할 수 있을 것 같습니다.
